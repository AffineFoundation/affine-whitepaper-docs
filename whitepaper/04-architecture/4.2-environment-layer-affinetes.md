# 4.2 The Environment Layer: Affinetes

**Zero developer burden.** Affinetes is a lightweight container orchestration framework designed specifically for evaluation environments. Its core design principle is that an environment author writes only business logic — the framework handles Docker image construction, HTTP server injection, deployment, networking, and cleanup.

**Two environment styles.** Affinetes supports two patterns for defining environments:

- **Function-based** (recommended) — the developer defines an Actor class with async methods and a Dockerfile. Affinetes automatically wraps this in an HTTP server layer, exposing all public methods as API endpoints. No server code required from the developer.
- **HTTP-based** — for environments that need custom routing or middleware, developers can provide their own server application. Affinetes detects this automatically and uses the developer's server directly.

**Three deployment modes.** Environments can be deployed in three ways depending on infrastructure requirements:

| Mode | What it does | When to use |
|---|---|---|
| **Docker** (default) | Manages containers with secure tunneled networking | Production evaluation on dedicated machines |
| **URL** | Connects to pre-deployed services at arbitrary HTTP endpoints | Environments already running on managed infrastructure |
| **Basilica** | Creates ephemeral Kubernetes pods per evaluation session | Cloud-native, auto-scaling deployment |

**Horizontal scaling.** A single environment can be deployed as multiple replicas across a heterogeneous set of hosts, with built-in load balancing across instances. This enables scaling of compute-intensive environments without changes to calling code.

**Lifecycle management.** The framework tracks all active environments and handles cleanup automatically — on normal exit, on unhandled exceptions, or on explicit request. Health checks detect container failures with automatic reconnection.
