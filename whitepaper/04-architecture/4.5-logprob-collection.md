# 4.5 Logprob Collection and Distributional Training Signals

**Per-token distributions.** Having established how environments and inference interact (Sections 4.2--4.4), this section describes an additional data channel that runs alongside standard evaluation. Beyond task-level reward signals, the system supports **per-token logprob collection** across multiple environments. When enabled via the `collect_logprobs` parameter, each inference call returns per-token log-probability distributions alongside the generated text.

**Dual-purpose data flow.** These distributions serve two purposes:

1. They power a dedicated **DISTILL** environment (detailed in Section 5.6) that measures distributional alignment between student models and teacher rollouts via KL divergence.
2. They feed the anti-copy detector's logprob signal (Section 3.5).

> **Key efficiency gain:** This dual-purpose data flow means that a single inference call — with no additional model queries — simultaneously supports evaluation scoring, training signal generation, and plagiarism detection.
