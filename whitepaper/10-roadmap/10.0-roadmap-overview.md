# 10. Future Roadmap

**Continuous evolution.** The Affine system is designed for continuous evolution. The directions outlined below represent areas of active exploration; given the pace of iteration across the project, priorities may shift as new opportunities or constraints emerge.

Development is organized along five axes:

1. **Model scaling** — expanding to larger architectures for greater RL capacity
2. **Distillation infrastructure** — making the teacher pipeline self-improving
3. **Inference independence** — reducing reliance on third-party inference providers
4. **Environment expansion** — broadening the capability axes and problem domains under evaluation
5. **Scoring refinement** — tightening anti-plagiarism and adaptive parameter tuning

**Table 16. Roadmap Summary**

| Axis | Current State | Direction | Key Dependencies |
|---|---|---|---|
| **Model Scale** | Qwen3-32B (fixed architecture) | Larger-scale base models | Higher-memory GPU instances; anti-copy threshold recalibration |
| **Distillation** | Single teacher, one-directional | Automated teacher selection; multi-teacher ensembling; corpus diversification | Miner quality surpassing current teacher |
| **Inference Infra** | Chutes (third-party) | Affine-operated or hybrid clusters | GPU procurement; endpoint migration |
| **Environments** | Current active suite | New problem domains; deeper existing axes; broader coverage | Environment development; scoring config updates |
| **Scoring** | Pareto + ELO + background anti-copy | Inline anti-copy integration; adaptive parameters | Population growth data; false-positive calibration |

> Each axis is detailed in subsections 10.1–10.5. The axes are independent but mutually reinforcing — for example, model scaling increases the value of inference independence, and environment expansion creates demand for scoring refinement.
