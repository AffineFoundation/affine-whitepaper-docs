# 7.1 Miner Workflow

**Four-step process.** A miner's workflow proceeds through four steps:

1. **Obtain a base model** from the network via `af pull`.
2. **Improve the model** through reinforcement learning or other training methods.
3. **Deploy the model** as a serverless inference endpoint on Chutes.
4. **Commit deployment metadata** -- including the Hugging Face repository, revision hash, and Chute identifier -- to the Bittensor blockchain. Each hotkey may hold exactly one active commitment at any time.

---

## Quick Start

The `af` CLI is the single entry point for all miner operations. To get started:

```bash
# Install the uv package manager
curl -LsSf https://astral.sh/uv/install.sh | sh

# Clone and install
git clone https://github.com/AffineFoundation/affine.git
cd affine
uv venv && source .venv/bin/activate && uv pip install -e .

# Verify installation
af --help
```

From here, the CLI provides commands for pulling base models, managing deployments, and committing on-chain. The full walkthrough — including API key configuration, model pulling, RL training tips, Chutes deployment, and on-chain commitment — is maintained in the [Getting Started guide](https://github.com/AffineFoundation/affine-cortex#getting-started) and [Miner documentation](https://github.com/AffineFoundation/affine-cortex/blob/main/docs/MINER.md).

## Architecture Constraint

To ensure fair comparison, all submitted models must conform to the **Qwen3-32B architecture** (hidden_size=5120, num_hidden_layers=64), and quantized models are rejected.

> **Why this matters:** This eliminates architectural confounds from evaluation -- score differences reflect training quality, not model capacity or quantization artifacts. The constraint also simplifies the anti-copy detector, which can compare hidden states and logprobs directly without accounting for architectural differences.

## Deploying on Chutes

Miners deploy their models as serverless inference endpoints on Chutes (Bittensor Subnet 64), which exposes them as OpenAI-compatible HTTP APIs at `https://llm.chutes.ai/v1`. The deployment provides:

- **GPU allocation** -- Chutes handles provisioning automatically
- **Request routing** -- standard chat completion requests from environments
- **Auto-scaling** -- scales with validator sampling demand

Environments invoke models by issuing standard chat completion requests to this endpoint. The miner is responsible for funding their Chutes account at a rate that keeps up with validator sampling.

## Committing On-Chain

The `af` CLI handles the on-chain registration process. The commit is a small metadata record containing:

- Hugging Face repo + revision hash
- Chute identifier

This record is registered against the miner's hotkey on Subnet 120. From this point, validators will sample the miner's model continuously.

## Watching the Dashboard

[affine.io](https://www.affine.io/) is the live view of the system. For each registered miner it shows:

- Per-environment scores
- ELO rating
- Current emissions
- Historical movement

The iteration loop is: **train -> deploy -> commit -> watch -> train**.
