# 5.4 Training Interface

The environment suite doubles as an RL training substrate. Every property that makes an environment suitable for scoring — verifiable reward, bounded compute, renewable task distribution — also makes it suitable for training. Affinetes exposes a gym-compatible `reset` / `step` / `stop` protocol over HTTP, so miners can run rollouts against the same container the scorer calls. This is where the evaluation-to-training loop closes.

### Deterministic Seeding

Every environment takes `(seed, task_id)` as initialization input and is required to produce identical trajectories for identical seeds. A miner running a rollout against SWE-Infinite task `swe-00123` with seed `42` observes the same repository state, the same test harness, and the same reward signal the scorer observed. Determinism lets a miner reproduce a scorer outcome locally when debugging, and lets RL methods that depend on consistent trajectories (PPO advantage estimation, for example) operate correctly.

### Reward Granularity Spectrum

The suite spans three reward granularities, and miners choose the one that fits their RL method. Dense per-step rewards are available on LiveWeb Arena and NavWorld — suited to algorithms that bootstrap value estimates from short horizons (PPO, A2C, online DPO variants). Per-task binary rewards on SWE-Infinite suit methods that tolerate sparse reward and long trajectories (REINFORCE with baseline, RLOO, GRPO). Per-token distributional rewards on DISTILL suit methods that distill from a reference distribution (KL-constrained policy gradient, KTO). MemoryGym and OpenSpiel admit both per-step and terminal decompositions depending on how the miner reads the episode.

### Same Artifact, Both Directions

Because the HTTP protocol is uniform, the container the miner trains against is the container the scorer calls. There is no "training fork" that diverges from the scored version over time. A new environment ships once; miners start training against it the same day the scorer starts using it. This property is what lets the market compound — a miner invests in learning the current surface, gets rewarded, and that reward funds the next investment against a surface that has meanwhile renewed ([§5.2](5.2-task-renewal.md)) without discarding prior learning.

**Table 5.4.** Reward granularity × environment × RL fit.

| Reward granularity | Environments | Example RL methods |
|---|---|---|
| **Dense per-step** | LiveWeb Arena, NavWorld | PPO, A2C, online DPO variants |
| **Terminal per-task (binary)** | SWE-Infinite | REINFORCE + baseline, RLOO, GRPO |
| **Per-token distributional** | DISTILL | KL-constrained policy gradient, KTO |
| **Terminal per-episode (graded)** | MemoryGym, OpenSpiel | Trajectory-level PG with shaped returns |
