# 5. Evaluation Environments

*The environment suite is under active development and evolves rapidly. The specific environments, configurations, and metrics described below reflect the state of the system at the time of writing and may change as new capability axes are added, existing environments are refined, or evaluation priorities shift.*

**Central design thesis.** Affine's environment suite is designed around a central thesis: meaningful evaluation of agentic intelligence requires interactive, renewable, and training-friendly environments — not static test sets.

Each environment is built to satisfy four principles:

- **Distinct capability axis** — every environment targets a different dimension of agent intelligence
- **Renewable task generation** — tasks are produced through an ongoing generative process, not drawn from a fixed pool
- **RL-compatible reward signals** — each environment provides reward signals structured for reinforcement learning
- **Complementary coverage** — five interactive environments evaluate behavioral capabilities, while a sixth (DISTILL) evaluates distributional alignment at the token level, targeting internal representation quality rather than task-completion behavior

> **Why this matters:** Static benchmarks saturate, leak, and reward memorization. Renewable, interactive environments ensure that evaluation remains meaningful as models improve and training data expands.

The table below summarizes the six environments; the subsections that follow detail each one.

**Table 5. Evaluation Environment Overview**

The following table maps each environment to the capability it targets, the size and nature of its task space, how new tasks are generated, and what training signal it provides. When scanning the table, note that all five interactive environments produce step-level or shaped rewards — not just binary pass/fail — while DISTILL provides a continuous distributional score.

| Environment | Capability Target | Task Space | Renewal Mechanism | Training Signal |
|---|---|---|---|---|
| **SWE-Infinite** | Software debugging | Unbounded (continuous) | Auto-discovery from GitHub PRs | FAIL_TO_PASS / PASS_TO_PASS tests |
| **LiveWeb Arena** | Browser-grounded agency | 197M+ configurations | Dynamic real-time data + combinatorial templates | Step rewards (exploration, efficiency) + terminal |
| **MemoryGym** | Memory management | 30--120 entities/eval x 10 domains x seeds | Seed-based generation + eval_salt perturbation | 4-axis scoring; shaped RL rewards |
| **QQR (NavWorld)** | Tool-mediated planning | 10,000+ tasks | 7 types x 3 difficulties x 71 cities x weekly salt | Per-step tool quality + 100-pt final |
| **GAME (OpenSpiel)** | Strategic reasoning | >10^60 trajectories | 22 games x config variants x seeds | Normalized game outcome [0,1] |
| **DISTILL** | Distributional alignment | Self-expanding (continuous) | Teacher rollout pipeline from raw corpus | exp(-KL) continuous score [0,1] |
