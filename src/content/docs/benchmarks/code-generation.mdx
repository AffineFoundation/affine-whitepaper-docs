**Code-centric evaluation.** Building on the stable and snapshot benchmarks in Sections 9.1-9.2, a separate evaluation covers code-centric benchmarks, providing direct evidence of whether Affine's SWE-Infinite training environment ([Section 5.1](/environments/swe-infinite/)) translates to external software engineering evaluation.

## SWE-rebench Results

SWE-rebench evaluates end-to-end software bug repair -- the same capability that SWE-Infinite is designed to train. Key findings:

- The **base model scores 0.00** -- unable to complete any repair tasks.
- **Leary CX achieves 12.28** and **Leary CS achieves 7.02**, demonstrating that Affine training produces measurable SWE capability where the base model has none.
- Axon1 remains at 0.00, consistent with the pattern of training-strategy-dependent outcomes observed across other benchmarks.

> **Why this matters:** This is the most direct evidence that Affine's environment-specific training produces transferable skills -- a base model with zero SWE capability gains non-trivial bug-repair ability through Affine training.

## HumanEval Results

HumanEval measures function-level code generation across 164 programming problems. Key findings:

- **Both Leary models surpass the base:**
  - Leary CX at 89.02 (+8.9% relative)
  - Leary CS at 88.41 (+8.2% relative)
- **Axon1 regresses** to 72.56, an 11.2% decline -- reinforcing that its training strategy sacrificed code-generation breadth.

## Tau2-bench Results

Tau2-bench (separate run from [Section 6.2](/scoring/anti-plagiarism/)) shows a different pattern:

- **Leary CS leads** at 21.43
- **Axon1** at 21.34 -- both above the base model's 14.05
- **Leary CX scores 7.76**, illustrating that no single miner dominates all axes

> **Key takeaway:** The Tau2-bench results reinforce that different training strategies produce different capability profiles -- strength on one code benchmark does not guarantee strength on another.
