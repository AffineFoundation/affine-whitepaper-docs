---
title: "8.2 Reward Signal Design Across Environments"
---

**Signal granularity spectrum.** Affine's environments provide reward signals at varying granularity, enabling miners to choose the RL training approach that best suits each environment's signal structure. The following subsections walk through the spectrum from densest to sparsest.

---

## Dense Per-Step Rewards

**LiveWeb Arena** and **NavWorld** provide rewards on every action within an episode.

**LiveWeb** rewards and penalties:

- +0.05 per new domain
- +0.06 per new asset
- +0.10 for target assets
- -0.04 for revisits
- -0.06 for blocked URLs
- +2.00 terminal bonus for >=80% accuracy

**NavWorld** scores each tool call based on a weighted formula:

> **What this means:** each individual tool call is evaluated on three dimensions, giving the model immediate feedback on both tool selection and usage quality.

```
tool_score = 0.4 * tool_selection + 0.3 * argument_quality + 0.3 * result_usefulness
```

> **Why this matters:** These dense signals make policy gradient methods tractable from early training, since the model receives learning signal at every step rather than only at episode completion.

## Per-Task Binary/Shaped Outcomes

Moving along the granularity spectrum, these environments provide feedback at the task level rather than per-step:

- **SWE-Infinite** -- binary reward signal: does the model's patch make the FAIL_TO_PASS tests pass without breaking PASS_TO_PASS tests?
- **MemoryGym** -- offers both modes:
  - **Binary mode:** correct = +1, wrong = 0
  - **Shaped mode:** store write = +0.3, correction flow = +0.5, correct answer = +1.0

## Per-Position Distributional Signals

**DISTILL** provides the smoothest gradient in the spectrum:

> **What this means:** the exponential KL mapping gives a continuous signal where every incremental improvement in distributional alignment produces a detectable score increase -- unlike binary task rewards where small improvements may not cross the pass threshold.

```
score = exp(-avg_kl)
```

## Game Outcomes

**OpenSpiel** returns a normalized score in [0, 1] per game. Strategic reasoning improvement is measured across:

- 22 games spanning seven quality tiers
- Trajectory diversity exceeding 10^60

## Deterministic Seeding

All environments accept deterministic seeds, enabling reproducible training episodes. The guarantees are:

- Given the same seed, task ID, and model, an environment produces **identical evaluation trajectories** -- essential for both fair scoring and training reproducibility.
- MemoryGym's `eval_salt` parameter perturbs entity attribute values between training and evaluation to prevent memorization of specific data points.

> **Key implication:** Deterministic seeding is what makes Affine environments usable as RL training substrates, not just evaluation benchmarks -- miners can replay episodes with different policies and observe the effect of each change.
