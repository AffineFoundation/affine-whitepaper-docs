Six environments are currently in production, each selected to force a distinct capability axis. Overlap is a cost: the Champion Challenge pipeline ([§3.2](/mechanism/scoring/)) counts each environment independently, so the suite is curated to span, not replicate.

### SWE-Infinite

Software engineering capability measured against real GitHub pull requests. An ingest pipeline mines the open-source commit stream, extracts pre-PR and post-PR repository states, and turns every PR into a task: given the pre-PR repo and issue description, the miner must produce a patch that makes FAIL_TO_PASS tests (failing before the PR) pass and leaves PASS_TO_PASS tests (passing before) still passing. Scoring is binary per task, so noise is pure Bernoulli and Pareto sample-size adjustments are straightforward. A five-machine validated-task cluster sustains ≥ 1,200 scored tasks/day.

### LiveWeb Arena

Browser-grounded web interaction through a headless browser. The environment combines live-web state with a combinatorial template system — entity slots, intent slots, action slots — producing ≈1.97×10^8 distinct task configurations. Each task specifies a goal ("find the cheapest direct flight from X to Y on Z and return the booking reference"); the miner issues browser actions (click, type, navigate, scroll) and receives step-level observations. Reward is shaped: small positive signals on constructive sub-actions plus a terminal bonus when the goal is achieved. Per-episode step and time caps prevent a single long trajectory from stalling a worker.

### MemoryGym

Memory management under a hard write budget. The environment generates worlds procedurally from a seed — ≈10 domain templates, each with its own entity vocabulary and correction-event schedule — and presents a sequence of write, update, and query events. The miner must maintain working memory, absorb corrections as they arrive, and answer queries with the most recently correct information. Scoring blends four axes (retrieval accuracy, correction compliance, write-budget adherence, latency-aware recall) into a composite. MemoryGym measures a capability orthogonal to single-turn reasoning, which is why it gets its own axis rather than folding into LiveWeb.

### NavWorld

Tool-mediated planning over a simulated travel graph. Six MCP tools are exposed (flight search, hotel booking, ground transport, weather, currency, itinerary check), and the task is to produce a trip plan for a given origin, destination set, and constraint bundle. The underlying world uses SHA256-seeded mock transport data with a weekly-rotating salt, so a miner cannot cache tool outputs week to week. Scoring is a 100-point rubric split between code-verifiable facts (dates, budgets, connection times) and LLM-judged quality (coherence, preference handling). 10,000+ tasks span 71 cities.

### OpenSpiel

Strategic reasoning across 22 games drawn from the OpenSpiel library — perfect-information, imperfect-information, simultaneous-move, and sequential. Each task is a full game played against a seeded opponent (fixed policies and self-play checkpoints), with the miner choosing moves through the environment's `step` interface. Scoring is the normalized outcome: win = 1.0, draw = 0.5, loss = 0.0, with intermediate shaping where games admit it. The effective trajectory space exceeds 10^60, which makes memorization of opponent policies infeasible.

### DISTILL

Distributional alignment measured as per-token KL divergence to a teacher distribution drawn from top-performing miners. Structurally, DISTILL is the odd one out — it is a pipeline, not a single service.

The pipeline runs three stages continuously. A **teacher worker** samples rollouts from the current top-K miners (ranked by ELO across the other five environments), captures token-level logprob distributions, and writes them into a shared reference pool. A **teacher mover** rotates, deduplicates, and salts the pool, writing batches into Cloudflare R2 buckets that the student scoring stage reads. **Student scoring** draws a prompt prefix, samples the candidate miner's token distribution on that prefix, and computes average per-token KL against the teacher. The final per-task score is $\exp(-\overline\{\mathrm\{KL\}\})$, which puts the score in [0, 1] and weights small divergences much more heavily than large ones. The teacher pool grows every time a new miner displaces the previous top-K, so the reference distribution is itself a function of the market.
