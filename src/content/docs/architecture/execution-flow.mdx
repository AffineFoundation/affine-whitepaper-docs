title: "4.6 Execution Flow Across Layers"
---

**Evaluation cycle.** With all three layers defined (Sections 4.1--4.5), the following walkthrough shows how they work together in practice. A complete evaluation cycle proceeds as follows:

1. The **coordinator** (affine-cortex executor) selects a batch of pending tasks from the task pool, each specifying a miner, environment, and task ID.

2. For each task, the executor invokes the target **environment instance** (managed by Affinetes) via HTTP, passing:
   - The miner's model identifier
   - The inference endpoint URL
   - Task parameters

3. The **environment** executes its evaluation logic — which may involve multi-turn interaction with the model. For each model query, the environment:
   - Issues an HTTP request to the **inference layer** (Chutes)
   - Receives the model's response
   - Continues execution

4. The environment computes a score based on the model's behavior and returns a `Result` to the executor.

5. The executor packages the result into a signed `SampleSubmission` and posts it to the backend API for storage and subsequent scoring.

> **Executor isolation:** At no point does the executor directly communicate with the inference layer; all model interaction is mediated through the environment. This keeps scoring logic encapsulated and the executor environment-agnostic.

---

**Teacher rollout pipeline.** A parallel pipeline handles **teacher rollout generation** for the DISTILL environment. This pipeline operates independently from the main evaluation cycle and involves the following steps:

1. An independent **teacher worker** process samples prompts from the Corpus-Eval environment.
2. The teacher worker runs a teacher model with logprob collection.
3. The teacher worker uploads rollouts to a **private R2 bucket**.
4. A separate **teacher mover** process periodically promotes a random subset of rollouts to a **public bucket**, where the DISTILL environment can access them for student evaluation.

See [Section 5.6](/environments/distill/) for full details on the DISTILL environment.
